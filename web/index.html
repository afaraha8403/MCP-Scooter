Installation

MCP Scooter is designed to be ultra-lightweight and run natively on your machine without Docker overhead.

# Windows (PowerShell)
irm https://mcp-scooter.com/install.ps1 | iex

For macOS and Linux, we provide pre-built binaries via GitHub Releases. You can also build from source using Go 1.24+ and Rust.

Identity Profiles

Profiles are the heart of Scooter. They allow you to isolate tools and credentials between work and personal environments, solving the "Configuration Chaos" of managing multiple account identities.

  • Isolation: No more manual key-swapping. Use one profile for personal tools (like Spotify) and another for work (like Jira/Postman) with independent keys.
  • Security: Credentials are stored in your native OS keychain (macOS Keychain, Windows Credential Manager).
  • Zero Drift: Configure once, use across all clients like Cursor, Claude, and VS Code.

Primordial Tools

Every AI client connected to Scooter sees these three tools by default. They enable **dynamic auto-choosing**—the LLM finds only what it needs for your specific question, avoiding context bloat and saving tokens.

scout_find(query)

Search for tools by name or capability across the registry.

scout_add(tool_name)

Instantly install and enable a tool for the current session.

scout_remove(tool_name)

Unload a tool to free up context window space and RAM.

MCP Registry

The registry defines how MCP servers are discovered and executed. Every tool in Scooter is a simple JSON manifest.

{
  "name": "brave-search",
  "version": "1.0.0",
  "title": "Brave Search",
  "authorization": {
    "type": "api_key",
    "env_var": "BRAVE_API_KEY"
  },
  "runtime": {
    "command": "npx",
    "args": ["-y", "@anthropic/mcp-github"]
  }
}

Security & Privacy

Scooter is local-first. Your data never leaves your machine unless you explicitly connect to a remote tool.

  • Local-First: No cloud dependencies. The server runs entirely on your hardware.
  • Keychain Integration: Tokens and API keys are stored in encrypted system vaults.
  • Human-in-the-loop: Sensitive actions (like file deletion or DB queries) require your manual approval.
  • WASM Isolation: Tools run in a secure sandbox, preventing them from accessing files outside their scope.

Hosting on GitHub

Yes! You can host your own MCP Scooter landing page or documentation site directly on GitHub Pages.

This site itself is built as a static HTML/Tailwind app, making it perfect for GitHub's native hosting. To deploy your own:

  1. Create a repository (e.g., my-mcp-docs).
  2. Place your index.html and assets in the root or a web/ folder.
  3. Enable GitHub Pages in Repository Settings.
  4. Set up a Custom Domain (optional) to point to your documentation hub.
